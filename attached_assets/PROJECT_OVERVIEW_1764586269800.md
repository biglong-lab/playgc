# 賈村競技體驗場實境遊戲系統 - 專案總覽

**作者**: Manus AI  
**日期**: 2025年12月1日  
**版本**: 1.0

## 專案簡介

本專案為**賈村競技體驗場**量身打造一套完整的實境遊戲開發與管理系統。此系統整合了現代Web技術與IoT硬體,讓遊戲設計者能夠輕鬆創建融合**射擊競技**、**實地探索**、**團隊協作**與**硬體互動**的沉浸式遊戲體驗。

### 核心價值

本系統解決了傳統實境遊戲開發的三大痛點:

1. **技術門檻高**: 透過視覺化編輯器,無需編程即可設計複雜的遊戲流程。
2. **硬體整合難**: 提供標準化的Arduino串接方案,讓實體設備與遊戲無縫整合。
3. **管理效率低**: 完整的後台管理系統,從遊戲創建到數據分析一站完成。

### 適用場景

- **戶外競技場**: 結合射擊、定位、任務的戰術遊戲
- **主題樂園**: 多點位的解謎探索體驗
- **團建活動**: 強調團隊協作的競賽遊戲
- **教育訓練**: 寓教於樂的互動式學習

## 系統特色

### 玩家端功能

玩家透過手機即可體驗完整的遊戲內容,主要功能包含:

- **多樣化頁面模組**: 故事敘述、角色對話、影片播放、互動按鈕
- **驗證與解謎**: 文字輸入、選擇題、條件分支等多種驗證方式
- **實境互動**: GPS定位導航、QR Code掃描、拍照任務
- **硬體整合**: 與射擊靶機、感應器等Arduino設備即時互動
- **社交功能**: 團隊即時聊天、位置分享
- **競技元素**: 即時積分、排行榜、成就系統

### 管理端功能

遊戲設計者與場域管理者擁有強大的後台管理工具:

- **視覺化編輯器**: 拖曳式頁面編排,所見即所得
- **資源管理**: 統一管理道具、事件、素材
- **設備監控**: 即時查看Arduino設備狀態與數據
- **數據分析**: 遊玩次數、完成率、熱門關卡等深度分析
- **測試工具**: 快速測試遊戲流程,發現潛在問題

## 技術架構

### 前端技術

- **React 18+**: 現代化的UI框架,組件化開發
- **Vite 5+**: 極速的開發體驗與優化的打包
- **Tailwind CSS + DaisyUI**: 快速建構美觀的響應式介面
- **Leaflet.js**: 開源地圖庫,顯示任務點與玩家位置
- **MQTT.js**: 實現與Arduino設備的即時通訊

### 後端服務

- **Supabase**: 提供完整的後端即服務(BaaS)
  - PostgreSQL資料庫
  - 即時資料同步(Realtime)
  - 身份驗證(Auth)
  - 檔案儲存(Storage)
- **HiveMQ Cloud**: 免費的MQTT Broker服務

### 硬體整合

- **Arduino (ESP32)**: 主控板,支援WiFi連線
- **MQTT協定**: 低延遲、高可靠的IoT通訊標準
- **感應器**: 紅外線、壓力、動作、RFID等多種感應器

### 開發與部署

- **Replit**: 用於快速原型開發與團隊協作
- **Vercel**: 可選的生產環境部署平台,提供更好的效能與穩定性

## 開發階段規劃

### Phase 1: 核心系統 (預計2週)

**目標**: 建立基礎架構與核心功能

- 專案初始化與環境設定
- Supabase資料庫建立與RLS策略
- 身份驗證系統(註冊、登入)
- 基本頁面模組(字卡、對話、按鈕)
- 遊戲大廳與場次管理

**交付成果**: 玩家可以註冊、登入,並體驗簡單的線性遊戲流程。

### Phase 2: 驗證與互動 (預計2週)

**目標**: 實作遊戲的核心互動機制

- 驗證型頁面(文字驗證、選擇驗證、條件驗證)
- GPS定位與地圖導航
- QR Code掃描功能
- 拍照上傳與簡易辨識
- 道具系統

**交付成果**: 玩家可以在戶外環境中進行定位、掃碼、拍照等互動,並完成解謎任務。

### Phase 3: Arduino整合 (預計2週)

**目標**: 實現線上線下的硬體整合

- MQTT通訊建立(前端與Arduino)
- 射擊靶機Arduino韌體開發
- 射擊數據即時回傳與計分
- 其他感應器整合(動作、RFID)
- 設備狀態監控

**交付成果**: 玩家射擊實體靶機後,分數即時更新到手機;管理員可監控設備狀態。

### Phase 4: 遊戲化功能 (預計1週)

**目標**: 提升遊戲的趣味性與競爭性

- 積分系統與計算規則
- 即時排行榜
- 團隊即時聊天
- 成就與徽章系統
- 遊戲結算與分享

**交付成果**: 完整的遊戲化體驗,玩家可以看到即時排名,與隊友溝通,並分享成績。

### Phase 5: 後台管理 (預計1週)

**目標**: 為管理者提供強大的管理工具

- 視覺化遊戲編輯器
- 頁面拖曳排序與跳轉設定
- 設備管理介面
- 數據分析儀表板
- 測試模式

**交付成果**: 管理者可以透過後台快速創建、編輯、測試遊戲,並查看數據分析。

### Phase 6: 測試與優化 (預計1週)

**目標**: 確保系統穩定性與使用者體驗

- 完整功能測試(單元測試、整合測試)
- 效能優化(圖片壓縮、資料庫查詢)
- UI/UX調整(根據測試回饋)
- 安全性檢查
- 正式上線準備

**交付成果**: 一個穩定、高效、安全的生產級系統。

## 風險與應對

### 技術風險

**風險**: Arduino設備在戶外環境的穩定性

**應對**: 
- 使用工業級的ESP32開發板
- 實作心跳機制,即時偵測設備離線
- 提供設備重啟與校正功能
- 預留備用設備

### 營運風險

**風險**: Replit免費方案的資源限制

**應對**:
- Replit僅用於開發與測試階段
- 正式環境部署至Vercel,確保穩定性
- Supabase提供充足的免費額度(50,000 月活躍用戶)
- 可視營運狀況升級至付費方案

### 使用者體驗風險

**風險**: 戶外環境的GPS定位不準確

**應對**:
- 設定合理的觸發範圍(10-20公尺)
- 提供手動確認機制
- 結合QR Code作為備用觸發方式
- 在遊戲設計時考慮定位誤差
