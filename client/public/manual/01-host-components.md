# 📺 HostScreen 軸線元件 — 14 個（大螢幕主視覺）

> 「群體投票答題搭配螢幕顯示」場景的所有元件
> 設計依據：[ADR-0004](../decisions/0004-host-screen-axis.md)

---

## 🎯 軸線特色

- **一對多單向廣播**：大螢幕主控、玩家手機端訊號上行
- **無隊伍、無人數限制**：50 人到 500 人都跑得動
- **無需登入**：玩家掃 QR 就加入（hostToken 認證）
- **使用場景**：園遊會、婚禮、頒獎、企業大會、講座、發表會

**運作流程**：
```
admin 後台建活動
    ↓
產生大螢幕網址 /host/:sessionId + 玩家 QR /play/:sessionId
    ↓
大螢幕（主持人筆電）打開、投影到舞台
    ↓
觀眾手機掃 QR 進入、即時互動上大螢幕
```

## 1. PollLive — 即時投票

| 項目 | 內容 |
|------|------|
| **pageType** | `host_poll_live` |
| **demo** | `/showcase` → poll-host |
| **特色** | 觀眾 5 秒內投完票、長條圖即時上升、可設多輪 |

**視覺**：大螢幕中央顯示題目 + 4 個選項長條圖 + 每選項投票數

**適用情境**：
- 頒獎典禮「人氣獎」由觀眾票選
- 企業內訓決策模擬「你會選哪個方案？」
- 講座中段「想多聽哪一塊？」
- 婚禮「祝新人幾歲生小孩？」

**admin 設定**：題目、選項（2-6 個）、倒數時間（10-60 秒）、是否多選

**玩家操作**：手機點選項 → 立即上傳 → 看到自己選擇

**出現在情境**：頒獎典禮、企業內訓

## 15. BingoBoard — 5×5 賓果集章板（W22 新增）

| 項目 | 內容 |
|------|------|
| **pageType** | `host_bingo_board` |
| **demo** | （Phase 6 W22 新增、demo 待補）|
| **特色** | 5×5 任務板、玩家完成任務即時打勾、連線達成（橫/直/斜）高亮 + 慶祝動畫 |

**視覺**：大螢幕 5×5 網格、完成的 cell 變綠 ✓、連線中的 cell 桃紅色發光、底部顯示「達成 N 條連線」

**適用情境**：
- 園遊會攤位集點（25 個攤位 = 一格一個）
- 商圈集章（25 家店、達成連線換獎）
- 員工旅遊任務板（拍照、打卡、找物 25 項）
- 生日 / 婚禮 Bingo（25 個情境如「找穿紅衣的賓客」）
- 親子冒險集章

**admin 設定**：
- 25 個 task（id + label + emoji + requiredCount）
- 標題 / 副標
- 連線達成獎勵規則（自訂）

**玩家操作**：
- 手機看任務列表
- 完成現場任務 → 點按鈕送 pulse
- 達 requiredCount 該 cell 變綠
- 看大螢幕同步連線進度

**連線計算**：
- 5×5 共 12 條線（5 橫 + 5 直 + 2 斜）
- 自動偵測達成條件
- 相同方陣才有斜線（5×4 等矩陣只有橫直）

**出現在情境**：通用補強元件、5 大市場全可用（推薦：園遊會、商圈、生日）

## 16. BlessingWall — 祝福瀑布牆（W22 新增、交誼類主視覺）

| 項目 | 內容 |
|------|------|
| **pageType** | `host_blessing_wall` |
| **demo** | （Phase 6 W22 新增、demo 待補）|
| **特色** | 祝福訊息從底部冒出、漂浮 8 秒消失、主題化色系（婚禮 / 生日 / 同學會 / 頒獎） |

**視覺**：大螢幕祝福訊息卡片漂浮飛起、配 emoji + 名字、主題色系隨情境變化

**適用情境**：
- 婚禮主視覺（玫瑰金 + 粉色）— 賓客留祝福、戒指交換時刻全場飄
- 生日壽星祝福（橙黃色）— 切蛋糕時刻全場應援
- 同學會重逢（紫藍色）— 久未見朋友各自寫近況祝福
- 頒獎得主應援（金色）— 得獎時觀眾為得主寫鼓勵

**admin 設定**：
- title / subtitle 標題
- theme 主題（wedding / birthday / reunion / awards / default）
- emojis 自訂可選 emoji 集（預設 8 個）
- maxLength 字數上限（預設 30）

**玩家操作**：
- 手機填表單（名字 + 祝福語 + 選 emoji）
- 送出後 throttle 3 秒（避免狂送）
- 訊息立即上大螢幕飄

**互補關係（vs GuestbookDigital）**：
- GuestbookDigital：靜態瀑布、永久顯示、適合活動全程留念
- BlessingWall：動態漂浮、瞬間情緒、適合高潮時刻

**出現在情境**：婚禮（核心）、生日、同學會、頒獎典禮（4 個交誼類情境）
