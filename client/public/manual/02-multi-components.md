# 👥 Multi 軸線元件 — 14 個（多人協作 / 對戰）

> 玩家手機端互動元件群、需 Firebase 登入 + 隊伍系統
> 設計依據：[ADR-0003 Squad 統一](../decisions/0003-squad-unification.md)

---

## 🎯 軸線特色

- **多人協作 / 對戰**：玩家組隊出戰（一場 = 一個 team；長期 = Squad）
- **Firebase auth**：玩家需登入、可累積戰績、跨場域沿用
- **Squad 系統**：「玩得不錯才保留」的長期身份、會累積記錄
- **使用場景**：街區走讀、企業內訓、員工旅遊、密室逃脫、親子冒險

**組隊流程（2026-05-04 重設計：「先玩、想留再留」）**：
```
玩家進活動 /team/:gameId
    ↓
看到三狀態 UI：
  - 沒長期隊伍 → 「+ 建新隊」「輸入別人組隊碼」
  - 有長期隊伍 → 「用 [大哉勇士] 出戰」+「+ 建另一隊」+「輸入碼」
    ↓
組隊完成 → 隊長按開始 → 全員進遊戲
    ↓
玩 → 結算
    ↓
隊長想保留？點「✨ 保留為長期隊伍」CTA
  - 是 → 取個名（如「公司勇者隊」）→ 升級為 Squad、戰績累積
  - 否 → 純 team、結束就解散
    ↓
下次再玩 → 看到「用 公司勇者隊 出戰」→ 1 click 出戰
```

**人數規則**：
- Squad（長期隊伍）規模 **不限人數**（10 人 / 50 人都 OK）
- 每場出戰人數依 `game.minTeamPlayers` ~ `game.maxTeamPlayers` 決定
- 一玩家最多保留 **5 個 Squad**（軟上限）
- Squad 內超過遊戲上限時：先到先進（按報到順序）

## 1. TreasureHunt — 尋寶任務

| 項目 | 內容 |
|------|------|
| **pageType** | `treasure_hunt` |
| **demo** | `/showcase` → treasure |
| **特色** | 線索式解謎、店家串聯、多人合作拼密碼 |

**視覺**：手機顯示題目 + 已找到線索清單 + 進度條

**適用情境**：
- 商圈打卡「集 5 家店線索拼密碼」
- 主題民宿故事「找線索解開保險箱」
- 親子冒險「跟孩子一起找寶藏」
- 校園歷史散步「追蹤校史線索」

**admin 設定**：題目、線索分布（地點 + 提示）、最終答案、獎勵

**玩家操作**：到指定地點 → 掃 QR / 輸入密碼 → 收集線索 → 拼出最終答案

**出現在情境**：商圈打卡、場域故事、親子冒險（核心元件）

## 2. GpsCascade — GPS 連鎖點解鎖

| 項目 | 內容 |
|------|------|
| **pageType** | `gps_cascade` |
| **demo** | `/showcase` → gps-cascade |
| **特色** | 強制走訪每一站、連鎖式解鎖、配合大螢幕地圖視覺化 |

**視覺**：手機顯示當前任務點 + 距離 + 已完成站數 / 總站數

**適用情境**：
- 街區走讀「金門後浦老街 8 站」
- 觀光局景點串聯「3 個景點集滿換紀念品」
- 校園歷史散步
- 主題樂園定點互動

**admin 設定**：站點座標 + 任務（拍照 / 答題 / 找物）、解鎖順序

**玩家操作**：移動到站點 → GPS 自動偵測 + 完成任務 → 解鎖下一站

**出現在情境**：街區走讀（公部門核心）

## 🆚 Multi vs Host 軸線快速判斷

| 你的場景需要... | 選哪條軸線？ |
|----------------|------------|
| 玩家需要登入 + 累積戰績 | Multi |
| 玩家需要組隊合作 | Multi |
| 玩家走訪實體場域（GPS）| Multi |
| 大螢幕 + 觀眾互動 | Host |
| 不需要登入、QR 即玩 | Host |
| 50-500 人同時參與 | Host |
| 場景包含多隊長期競爭 | Multi（用 Squad）|
| 場景是一次性活動 | Host（無紀錄壓力）|
