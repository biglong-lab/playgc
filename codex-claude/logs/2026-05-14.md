業主回饋：「碎片預設數量變為 0，但依舊無法進行設定」

### 第三輪（真根因）→ commit `78d384e2`

直接看 PageConfigEditor：

```ts
const updateField = (field, value) => {
  onUpdate({ ...config, [field]: value });  // ⚠️ config 是 closure stale
};
```

useEffect 連續 call updateField 3-4 次：
```ts
updateField("fragments", generated);       // → onUpdate({...config_stale, fragments: [...]})
updateField("fragmentCount", 4);           // → onUpdate({...config_stale, fragmentCount: 4})  ← 沒 fragments
updateField("fragmentType", "numbers");    // → onUpdate({...config_stale, fragmentType: "numbers"})  ← 都沒
updateField("targetCode", "1234");         // → ...
```

**每次 spread 的 config 都是 stale closure**、後面 onUpdate 覆蓋前面 → 只剩最後一個 field 真的寫入。

**這就是業主回報「碎片數量永遠是 0」的真根因。**

### 真正修法

1. **PageConfigEditor** 加 `updateFields(patch)` batch API：
   ```ts
   updateFields={(patch) => onUpdate({ ...config, ...patch })}
   ```
2. **ConditionalVerifyEditor** props 加 `updateFields?`
3. **useEffect** 內改用 batch：
   ```ts
   const patch = { fragments: generated };
   if (isFragmentCountDirty) patch.fragmentCount = hasCount;
   if (!config.fragmentType) patch.fragmentType = type;
   if (type !== "custom") patch.targetCode = generated.map(f => f.value).join("");
   updateFields(patch);  // 一次寫入、不會 stale
   ```
4. **updateFragments** 同步改 batch
