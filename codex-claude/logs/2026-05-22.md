## 已部署修改（commits bbabbf1d + 2f7c9c3f + fb7a4791）

### #11 保留進度三態（重新設計）

業主明確要的規則：
- **沒玩過 / 沒保留進度** → 顯示【開始遊戲】
- **有保留進度** → 顯示【返回遊戲】+【重新開始】
- **完成過** → 只顯示【再玩一次】

修改：
- `client/src/pages/GameBySlug.tsx`：
  - 從 `hasActiveProgress` 單一旗標改為 `sessionStatus` + `hasResumablePlaying` + `hasCompleted` 三態
  - 三態按鈕分支
  - 「重新開始」「再玩一次」帶 `?restart=1` query param
- `client/src/pages/GamePlay.tsx`：
  - `isReplayMode` 識別 `replay=true || restart=1`、進入後強制建新 session、不顯示 ResumeDialog

### #14 runtime 不再彈進度提示

根因：
1. `GamePlay.tsx` 有兩處 `<ResumeDialog>`：line 583 早期 return + line 789 冗餘
2. query refetch 把 existingSession 帶回來、就會再次 setPendingDecision(true)

修法：
- 移除 line 789 冗餘 ResumeDialog（只留 line 583 early return）
- `useSessionManager.ts`：setPendingDecision 條件加 `!state.sessionId`（已建 session 後永不再彈）

### #13 移除遊戲開始 toast
- `useSessionManager.ts` line 161：移除 `toast({ title: "遊戲開始", description: "祝你好運!" })`

### #3 字體大小切換無效（**根因發現**）

根因：`.game-prose` class **零個元件實際使用** → CSS 變數 `--reader-scale` 切換沒元素響應、看起來「無效」。

修法：
- `client/src/pages/GamePlay.tsx`：`<main>` 加 `game-prose` class、所有後代繼承
- `client/src/index.css` `.game-prose` selector 擴大繼承到 div / h1-h4 / button / label
- 系統 UI（app-header / player-bottom-nav / icon-button / floating-tool）即使在 `.game-prose` 內也強制 16px

### #4 <380px 上欄裁切

`client/src/components/shared/GameHeader.tsx`：
- header 外層 `max-w-full overflow-hidden`
- 內層 `px-2 sm:px-4` `py-2 sm:py-3` `gap-1 sm:gap-2`
- 左群組 `min-w-0 flex-1` + 標題 `truncate flex-1`
- 右群組 `flex-shrink-0` `gap-0.5 sm:gap-1`
- 所有 icon button `h-9 w-9 sm:h-10 sm:w-10`（窄裝置縮小 36→40）

### #9 按鈕長文字超出

`client/src/components/game/solo/ButtonPage.tsx` line 175：
- text span 加 `break-words whitespace-normal min-w-0`
- icon `flex-shrink-0`
