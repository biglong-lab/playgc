### 3. PWA 自動恢復韌性（commit `f0caa72b`，已部署）
- **根因①**：手動 `docker compose --build` 漏帶 GIT_SHA → `/api/version` 回 `unknown` → 前端版本比對（serverCommit==="unknown" 直接 return）整個失效
- **根因②**：一次性恢復 flag → 首次 reload 沒成功就永久卡死、重整半天不恢復
- **根因③**：Safari React.lazy chunk 失敗的 `_result.default` TypeError 漏偵測
- **修**：恢復節流改 `canRecover` 時間窗（5 分窗最多 3 次）；`isChunkLoadError` 加 `_result` 樣態（`main.tsx` + `ErrorBoundary.tsx`）

- 加 `SSH_PORT="${SSH_PORT:-52099}"` 套用全部 4 處 ssh；以後 `npm run deploy` 一鍵跑通（含 GIT_SHA 注入）

## [Claude] 17:xx — 預約改時間「不在開放時段」修復（commit `a61ca718`，已部署）

### 背景
業主回報：預約 7/12 改 7/9 報「新時段不在開放時段內」，但單獨新增同時段成功。

### 根因
`field_id` 混用 UUID/code：`booking_configs.field_id='JIACHUN'`（代碼），但 `updateBooking({fieldId: scope.id})` 傳 UUID `72cc204d-…` → `getBookingConfig(UUID)` 用 `lower(field_id)=lower(UUID)` 比對 miss → `getAvailability` 回 `[]` → `slots.find` undefined → 拋 slot_not_open。新增走 `createManualBooking`（不查開放時段、直接 insert）故不受影響。

### 修法（首選：根治、不動資料/schema）
`getBookingConfig` 先用 `fields` 表把傳入值（UUID 或 code）正規化成 id+code，再一起比對 `booking_configs`。一處修、getAvailability/預約頁全受惠。

- tsc 0；生產 DB 模擬新查詢 `getBookingConfig('72cc204d-…')` → 找到 `JIACHUN`（啟用）
