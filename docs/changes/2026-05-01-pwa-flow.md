## 背景

使用者實機反映：
- 「PWA 用起來不夠順暢，要刻意才會使用」
- 「掃描 QR 後跳出瀏覽器，沒回到 PWA」
- 「在 PWA 內走錯場域沒地方回首頁」
- 「PWA 安裝提示太頻繁，按了取消又出現」

設計依據：[`archive/PWA_USER_FLOW_OPTIMIZATION_V2.md`](../archive/PWA_USER_FLOW_OPTIMIZATION_V2.md)（已 archived）

---

## 解決方案

把 PWA 流程拆 4 個 phase + 介面避讓 + 安裝提示防擾人，分開部署。

### Phase A — 全域救援動線
浮動回首頁鈕（`FloatingHomeButton`），左下角，PWA 鎖死狀態救援。

### Phase B — start_url + lastVisitedField 智能路由
- manifest `start_url` 加 `?launch=pwa` flag（給統計用）
- `lastVisitedField` localStorage 隔離 active/cached fieldCode
- SmartRedirect 優先走 lastVisitedField

### Phase C — PWA 內 QR Scan FAB
`InAppQrScanFAB` 元件 — Home 頁卡片「📸 掃描 QR Code 進入遊戲」，內掃同站 URL → in-app navigate（不離開 PWA）。

### Phase D — 使用情境統計
- `client/src/lib/pwa-analytics.ts`（client log）
- `client_events` 表（既有，無需新 schema）
- `GET /api/admin/analytics/pwa-usage`（admin endpoint）
- `/admin/pwa-analytics` 後台頁（折線圖 + 分布條）

### 介面避讓 + 防擾人
- 工具類 `.pb-bottom-nav` 涵蓋 PlayerBottomNav 高度 + safe-area
- Home / Leaderboard / MeCenter / BattleLayout sticky header 加 `safe-top`
- PWAInstallPrompt：dismiss cooldown 7 天 → 30 天 + 加「不再提示」永久關閉 + 累計第 3 次訪問才彈
- PwaInstallEntry 主動入口（在「我的」頁面）— iOS 顯示分享教學 dialog
