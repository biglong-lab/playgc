## 📊 預期指標

| 指標 | 目標 |
|------|------|
| Phase 3 結束時客戶數 | ≥ 3 場真實付費 |
| 月訂閱客戶 | ≥ 1 個（民宿 / 親子館）|
| AI 內容 fallback 率 | < 10%（90%+ 的內容能被 admin 直接用）|
| 付費成功率 | > 95% |
| API 代理商 | ≥ 1 個技術整合 |
| 累計營收（4 週）| NT$ 30,000-50,000 |

---

## 🛡 紅線（不可違反）

1. **付費失敗不影響既有流程**：客戶卡關時 admin 可手動建場
2. **AI 內容必須有 fallback**：DeepSeek 失敗 → 用既有 default 範例
3. **API 必須有 rate limit**：避免代理商濫用 / 攻擊
4. **發票合規**：每筆台幣付費都要開立電子發票（接 ezpay / 智付通）
5. **資料隔離**：API 取得的資料只屬於該 API key 的場域
