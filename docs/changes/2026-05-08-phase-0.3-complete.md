## 4. 爭議仲裁範例（業主可用）

### 場景：媽媽問「我兒子那場明明答對 q3、為什麼沒分數」

**Admin 操作流程**：

1. 進 `/admin/multi-sessions` 找到當天該 session
2. 點 「Replay」按鈕
3. 進 Replay 頁面
4. 篩選：玩家 = 兒子 user
5. 找 q3 時間軸範圍（如 14:25 ~ 14:26）
6. 看到事件：
   - `14:25:30.123 message inbound team_score_update payload={questionId:"q3", choice:2}`
   - `14:25:30.124 broadcast outbound team_score_error payload={reason:"invalid_choice_index", maxChoice:3}`
7. 結論：兒子在 client 端點到非 4 個選項之外的座標、計分系統正確拒絕
8. 點「CSV 匯出」存檔給家屬看時間戳

**證據級紀錄**：
- 含完整 payload（client 送的 choice、server 回的 reason）
- 含時間戳（毫秒精度）
- 含 clientIp（誰的設備）
- 業主 / 家屬都能看時間軸有依據
