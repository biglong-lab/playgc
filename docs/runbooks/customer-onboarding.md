# Runbook — 業務客戶 onboarding 完整 SOP

> 給業務 / 主辦方接洽用
> 適用：第一次帶客戶從零到第一場活動上線
> 時間：30-60 分鐘（不含活動本身）

---

## 🎯 為什麼要這份 SOP

Phase 3 W9 啟動：找第一個付費客戶。
業務帶客戶時最怕：
- 客戶問問題答不出來
- 帶錯動線（先看 Pitch 還是直接開 Wizard？）
- AI 預覽生成內容不適合
- 現場掃 QR 不能用

本 SOP 是「**保證能帶客戶從不認識到付費**」的 step-by-step 流程。

---

## 📞 Step 1：接洽 / 預約（10 分鐘）

### 開場白（LINE / 電話 / Email）

> 「您好！我是 CHITO 數位遊戲平台的 [姓名]。
> 看到您可能要辦 [婚禮/破冰/內訓/...]，
> 我們有一個能讓你的活動瞬間升級的工具 — 預組好 12 種情境，10 分鐘搞定建場 + 印好 QR。
> NT$ [3,000-15,000] 包含完整建場 + 後續調整。
> 願意花 30 分鐘聊聊嗎？」

### 預約前確認

- 對方場合（婚禮 / 破冰 / 內訓 / ...）
- 預估參與人數（< 30 / 30-100 / 100+）
- 活動日期（決定我們有多久準備）
- 場地（有沒有投影機、Wi-Fi 是否穩）

### 答應接洽 → 寄行前資訊

> 「好的！我們約 [時間]。
> 在會面前可以先看一下我們的平台介紹：https://game.homi.cc/pitch
> 會面時我會帶你跑一次完整流程。」

---

## 🎤 Step 2：會面 / 帶看（20 分鐘）

### 動線

```
1. 開 https://game.homi.cc/pitch
   ↓ 帶看 7 個區段（10 分鐘）
2. 開 https://game.homi.cc/find-scenario
   ↓ 客戶答 3 題（2 分鐘）
3. 看 Top 3 推薦
   ↓ 點 🥇 進詳情頁（1 分鐘）
4. 詳情頁 → 帶客戶看「適用情境」+ 「商業價值」
   ↓ 客戶確認要這個（5 分鐘）
5. 客戶決定 → 收費 / 付款（看付費機制狀態）
```

### Pitch 帶看重點（10 分鐘）

依序帶看 7 區段：

1. **痛點**：「氣氛不對、沒互動、沒紀念」 → 客戶點頭
2. **解法**：3 軸線（大螢幕 / 隊伍 / 紀念）→ 對應客戶場合
3. **12 情境**：點客戶分類（婚禮 / 內訓 / 街區 / ...）→ 帶看實例
4. **5 步驟流程**：強調 10 分鐘搞定 vs 過去 1 小時
5. **收費三方案**：依客戶規模指相應方案
6. **對比表**：強調我們不是「自己做」也不是「客製外包」 — 平台化
7. **CTA**：「我帶你試一下 3 問找情境」

### Find Scenario Wizard 帶看（2 分鐘）

直接讓客戶在你的螢幕上點：
- Q1 分類
- Q2 人數
- Q3 重點

看 Top 3 推薦 → 客戶通常會選 🥇

### 詳情頁 → 客戶決定（5 分鐘）

帶看詳情頁的：
- 完整 description
- 適用情境（驗證匹配）
- 元件組合（讓客戶看到具體會用什麼）
- 商業價值（強調你的收費合理）

問客戶：「這樣的組合對你的活動有用嗎？」
→ 如果客戶「有用」 → 進 Step 3
→ 如果客戶「不太對」 → 回 Step 2 Pitch 改用另一個情境

---

## 💰 Step 3：收費 / 簽單（10 分鐘）

> Phase 3 W10 才會做付費機制。W9 階段先用「先付款後建場」LINE Pay / 轉帳。

### 收費標準（從 PitchDeck 對應）

| 場合 | 範圍 | 建議 |
|------|------|------|
| 小型婚禮（< 50 人）| NT$ 5,000-8,000 | NT$ 6,000 |
| 中型婚禮（50-150 人）| NT$ 8,000-15,000 | NT$ 10,000 |
| 破冰熱場（< 80 人）| NT$ 5,000-12,000 | NT$ 8,000 |
| 員工旅遊（< 100 人）| NT$ 10,000-30,000 | NT$ 15,000 |
| 內訓 SaaS 訂閱 | NT$ 1,500-5,000 / 月 | NT$ 2,500 |

### 包含項目（必須事前說清楚）

✅ 完整建場（含 12 情境模板任選 1）
✅ AI 客製內容（含新人姓名 / 活動主題）
✅ A4 QR 列印 PDF
✅ 活動前 1 次調整內容
✅ 活動當天 LINE 即時技術支援

❌ 不含：投影機 / Wi-Fi / 現場人員

### 收款方式（W9 階段過渡）

LINE Pay / 銀行轉帳 / 現金（預付 50% / 完成 50%）

W10 完成後 → 改用 Stripe Checkout（自動化）

## 🛠 Step 4：建場（admin 操作，5 分鐘）

收款後：

1. admin 登入 https://game.homi.cc/admin
2. 點 dashboard 上的 🌟 「情境模板市集」入口
3. 進客戶選的情境詳情頁
4. **填 AI 客製內容**：
   - context 範例：「Hung 與 Anita 5/15 晶華婚禮、新娘喜歡櫻花、賓客約 100 人」
5. 點「預覽 AI 客製內容」 → 等 5-10 秒
6. 看 ✅/⚠ 元件清單 → 確認 OK
7. 點「用 AI 內容建場」 → 等 5 秒
8. 結果 Dialog 點「列印 QR」
9. A4 印出（每張 QR 一頁）

### 失敗 fallback

| 問題 | 解法 |
|------|------|
| AI 預覽失敗（503）| 場域 OpenRouter API key 未設定 → 用「default 範例建場」 |
| AI 內容不適合 | 改 context 重試、或用 default + 後續手動編輯 |
| 建場 401 | admin session 過期 → 重新登入 |

```
[hostUrl]

[QR PDF 附件]

- 大螢幕網址含 hostToken 12 小時有效
- 玩家 QR 公開展示沒問題
- 活動前 1 天請通知我做最終確認
- 活動當天若有問題可立即 LINE 我

📞 我的聯絡方式：
[姓名 / LINE]
```

## 📅 Step 6：活動當天 + 後續（持續）

### 活動前 1 天

- LINE 客戶確認：「明天活動，需要調整內容嗎？」
- 若需要 → 重新跑 AI 預覽 + 建場（給客戶新 URLs）

### 活動當天

- 開 LINE 待命
- 確認客戶到場、QR 已貼、投影機正常
- 若異常 → 即時排查（多半是 hostToken 過期 → 重建場）

### 活動後

- LINE 客戶：「活動順利嗎？要不要我把所有留言 / 拍立得整理寄給你？」
- 寄活動回顧（從 host_session state 中提取留言）
- 邀請：「下次活動再找我！」/「介紹給朋友？」
- 索取見證 / 案例授權

---

## 🚨 紅線（不可違反）

1. **不誇大**：客戶問「是否一定不會故障？」 → 答「99% 穩定，異常我即時處理」
2. **不亂報價**：依規模對應 PitchDeck 區間
3. **不繞過付款**：先收款再建場（避免欠款）
4. **不外洩 hostToken**：玩家端 QR ≠ 大螢幕端 QR、不混用
5. **不刪除既有資料**：客戶活動結束後保留 7 天，再依情況歸檔
